§ 02

规律总结 · 结构原型白描

Patterns & Archetypes

五种屏幕机械结构原型(工业设计白描)

翻转铰链细节(主流增量方案的工程收敛)

八条统一规律

屏幕尺寸呈「双峰」分布,6″ 附近真空

副屏档集中在 1.9–5″(信息显示/轻触控),一体交互档集中在 5.5–8″(完整 UI),10″+ 属于「屏幕即产品」的终端类。6″ 档几乎无人做——小于完整交互、大于仪表显示,定位尴尬。

1280×800 / 1280×720 是绝对主流水位

18 款中 6 款直接落在 1280 长边(5″–10.1″ 全覆盖),源自平板/车机屏供应链;副屏用低分异形比例(960×400、480×854、170×320),4K 副屏(Miniproca)是营销炫技而非趋势。

材质:IPS LCD 一统,触控=电容多点标配

除 Lenovo 旗舰用 OLED 外,全部为 IPS LCD;凡定位「交互」的屏幕均为电容触控(10 点),仅 1.9–3.5″ 仪表屏保留纯显示。玻璃盖板 + 全贴合是 4″ 以上的默认工艺。

结构演化:固定 → 翻转 → 电机

2015–23 年产品几乎全为固定结构;2024–26 年新品(G-Flip、Miniproca、ViewDock、AM01S、Auto Twist)集中转向可动结构。翻转铰链借 GBA-SP 的心智成为增量共识;电机方案仅 Amazon/Lenovo 两家大厂敢碰(成本、噪音、可靠性、桌面空间)。

翻转铰链参数已收敛

铰链一律在顶面前缘;开角 0–65°/80°/90°;铝合金铰链座 + 摩擦扭矩轴无级悬停;合拢时屏幕面朝下、顶面即保护盖。这套参数可直接作为 Agent PC 的工程基线。

副屏的「系统归属」分裂,须尽早定义

UH185/GEM12/S1 的副屏由独立 MCU/固件驱动,不显示主系统画面(UH185 明确无法显示 Windows);AM01S 反其道把「原生横屏、可作系统扩展屏」当核心卖点。Agent PC 必须在架构期决定:副屏归 Agent 运行时,还是归主 OS。

摄像头·灯光·语音三件套几乎空白

18 款中摄像头仅 3 款、远场语音仅 3–4 款、语义化灯光仅 2 款(S1 的 RGB、Violoop 的底部氛围光,均无语义);三者齐备的迷你主机为 0。智能音箱有语音无算力,迷你主机有算力无感知——这正是 Agent PC 的差异化窗口。

Footprint 收敛,堆叠已有先例但顶面被抢占

带屏主机收敛于 138–175mm 方形 footprint,与 Mac mini(127)/DGX Spark(150)同带;DGX Spark 官方支持双机成簇。矛盾点:翻转屏、无线充都在抢顶面,而 Mac mini 类靠顶/底面呼吸——可堆叠设计必须把风道转移到侧后向。

§ 03

Agent PC 推荐 · 形态学收敛

Recommendation

你的需求约束:触控屏 + 摄像头 + 灯光 + 语音交互,且算力盒可能像 Mac mini 一样多台并行、需要考虑堆叠收纳。先穷举规格空间回答「有多少种可能」,再按约束收敛。

诚实的风险提示(反方视角)

① 别轻易上电机。Echo Show 10 要求 38×25cm 旋转净空,中国/日本桌面场景吃紧;Lenovo 电机铰链把整机推到 $1,649 且噪音/耐久尚未被市场验证。人脸「追踪感」可以用 13MP 广角+数字裁切以 1/10 成本实现 80% 效果。
② 副屏系统归属必须在架构期定死。走独立 MCU(UH185 式)则响应快、休眠可显,但 Agent UI 迭代受固件拖累;走主系统扩展屏(AM01S 式)则生态灵活,但开机前/死机时屏幕是黑的——Agent 设备恰恰最需要「主机挂了屏幕还能说话」的兜底,建议 MCU 常显层 + 主系统投射层 双层架构。
③ 堆叠与散热天然打架。塔式堆叠盖死了每层顶面,而样本中所有 65W 级机型都依赖顶部或底部呼吸。若坚持可叠,风道必须全侧向,并对 3 层以上做热仿真;否则学 DGX Spark 只承诺双机成簇,把「多机」交给竖插托架。
④ 供应链与品控。4K 小屏(Miniproca)单价高、驱动功耗大,勿被参数带偏;众筹小厂(ViewDock 初代)已出现发货与连接投诉——铰链供应商选型和 2 万次寿命测试要放进 EVT 关键路径。
⑤ 常开摄像头的信任问题。Agent PC 的摄像头是「常开传感器」而非视频会议配件,物理滑盖(而非电子开关)+ 灯环状态联动是目前唯一被用户信服的方案,应作为不可删减项。
